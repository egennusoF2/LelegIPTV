## Build and install

Pick the artifact that matches your device. LelegIPTV documentation points to
local build outputs from this repository, not to official app-store listings.

| Platform | Build command | Artifact path |
|---|---|
| Windows 10 / 11 | `pnpm tauri build` on Windows | `src-tauri/target/release/bundle/nsis/*.exe`, `src-tauri/target/release/bundle/msi/*.msi` |
| macOS | `pnpm tauri build` on macOS | `src-tauri/target/release/bundle/dmg/*.dmg`, `src-tauri/target/release/bundle/macos/*.app` |
| Linux | `pnpm tauri build` on Linux | `src-tauri/target/release/bundle/deb/*.deb`, `src-tauri/target/release/bundle/rpm/*.rpm`, `src-tauri/target/release/bundle/appimage/*.AppImage` |
| Android phone / tablet / TV | `pnpm tauri:android:build` | `src-tauri/gen/android/app/build/outputs/apk/**/*.apk`, `src-tauri/gen/android/app/build/outputs/bundle/**/*.aab` |
| Chromebook / Android XR | `pnpm tauri:android:build` or `pnpm build` | Android artifacts or `dist/` depending on deployment |
| iOS / iPadOS | `pnpm tauri:ios:init`, then `pnpm tauri:ios:build` on macOS | `src-tauri/gen/apple/build/**/*.xcarchive`, exported `.ipa`, or the local Xcode Organizer export path |
| Samsung Tizen TV | `pnpm build && pnpm tizen:prepare`, then Tizen Studio/CLI package | Prepared root: `build/tizen-web/`; signed package: `*.wgt` |
| Web browser | `pnpm build` | `dist/` |

Desktop artifacts are host-platform specific: build Windows packages on
Windows, macOS packages on macOS, and Linux packages on Linux unless you
maintain a dedicated cross-compilation setup.

### macOS: "LelegIPTV.app" cannot be opened

The macOS build is not yet notarized by Apple, so Gatekeeper blocks it on first launch with a message like *"Apple could not verify LelegIPTV.app is free of malware"*. After dragging the app from the `.dmg` into `/Applications`, remove the quarantine flag from a Terminal:

```bash
xattr -dr com.apple.quarantine "/Applications/LelegIPTV.app"
```

Then open the app normally. You only need to do this once per install.

<Callout type="tip">
  This is a one-time step until a notarized macOS build ships. The command is safe: it strips the `com.apple.quarantine` extended attribute that Gatekeeper sets on downloads, nothing more.
</Callout>

## First launch

1. Open the app. You'll land on the login screen.
2. Paste your **Server URL**. Include the port if your provider gave you one.
3. Fill in **Username** and **Password** if your provider gave you separate credentials. If they only gave you a single M3U URL, leave these blank.
4. The app connects, loads your channels, and takes you to the home screen.

You only need to sign in once. The app remembers your details for next time.

<Callout type="tip">
  More on setting up Xtream credentials, M3U URLs, and multiple playlists in [Playlists (Xtream + M3U)](./playlists/).
</Callout>

## What's on the home screen

- **Live TV** - your channel list, with categories and what's on now / next.
- **Movies** - poster grid with category filtering.
- **Series** - poster grid with seasons and episodes inside each show.
- **Continue watching** - a strip of what you were last watching, picked up from where you left off.
- **Sidebar** on desktop, or a **bottom tab bar** on phones, for jumping between Live TV, Movies, Series, Favorites, Watchlist, Downloads, Search, EPG, and Settings.
